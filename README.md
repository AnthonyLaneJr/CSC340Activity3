# CSC340Activity3
CSC 340 Activity 3

This GitHub Repository Contains 9 blueprints detailing the tables created to represent the databse from Activity 2.
The Blueprints will be individually listed below along with their purposes and what information they show.

LandingPage:
The Landing page is the homepage to the application and provides brief information on all transactions completed. This can be thought of as the view all page for transactions. From here the user can view each transcation and navigate to individual transaction pages (by clicking transaction ID), create new transactions, navigate to the Expenses page, and navigate to the Employees page. 

IndividualTransactionBlueprint:
The page for viewing indivual transaction entry information. From here the user can access the link to edit the transaction info, delete the transaction entry, or navigate back to the landing page.

NewTransactionBlueprint:
This is the page that the user is routed to if they click the "add new," button on the Transcation page blueprint. Alternatively the user is also routed here if they hit the "edit," button on the individual transaction blueprint page. From here the user can input or edit a transcation entry's information. They have a save button and a discard button on this page, as well as a link back to the landing page. For transactions, many of the field are auto generated at the time of entry, so not all fields are inputtable upon creating a new transaction.

EmployeePageBlueprint:
The Employee Page blueprint is the page in which all employess can be viewed (shows the Emplyee table). From here the user can view all employees, click the top link to return to the landing page, click the add new button to add a new employee, click the X button next to each employee to delete said employee, or click the employee ID of a specific entry to view that employess individual entry.

IndividualEmployeeBlueprint:
The page for viewing indivual employee entry information. From here the user can edit the employee info, delete the employee entry, or navigate back to the landing page.

NewEmployeeBlurprint:
This is the page that the user is routed to if they click the "add new," button on the employee page blueprint. Alternatively the user is also routed here if they hit the "edit," button on the individual employee blueprint page. From here the user can input or edit an employee entry's information. They have a save button and a discard button on this page, as well as a link back to the landing page.

ExpensePageBlueprint:
The Emxpense Page blueprint is the page in which all Expense Types can be viewed (shows the Expense table). From here the user can view all Expenses, click the top link to return to the landing page, click the add new button to add a new Expense, click the X button next to each Expense to delete said Expense, or click the Expense ID of a specific entry to view that Expenses's individual entry.

IndiviudalExpenseBlueprint:
The page for viewing indivual Expense entry information. From here the user can edit the expense info, delete the expense entry, or navigate back to the landing page.

NewExpenseBlueprint:
This is the page that the user is routed to if they click the "add new," button on the Expense page blueprint. Alternatively the user is also routed here if they hit the "edit," button on the individual Expense blueprint page. From here the user can input or edit an Expense entry's information. They have a save button and a discard button on this page, as well as a link back to the landing page.
